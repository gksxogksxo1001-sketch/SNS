# [PRD] 사용자 인증 (회원가입 및 로그인)

## 1. 개요 및 목적
서비스 이용을 위한 최소한의 개인정보를 수집하고, 보안이 강화된 인증 시스템을 통해 사용자 데이터를 보호하며 편리한 접근을 제공합니다.

## 2. 타겟 사용자 프로파일
- 신속한 회원가입과 간편한 로그인을 선호하는 2030 모바일 네이티브 사용자
- 개인정보 보안과 소셜 계정 연동의 편리함을 동시에 추구하는 사용자

## 3. 핵심 사용자 여정 (User Flow)
1. **로그인 페이지** 진입
2. **소셜 로그인**(Google, Kakao, Naver 등) 또는 **이메일 로그인** 선택
3. 이메일 가입 시: 이메일 입력 -> 비밀번호 설정 -> **이메일 인증** 진행
4. 로그인 성공 후 **성향 설정(프로필 세팅)** 페이지로 이동
5. 비밀번호 분실 시 **비밀번호 재설정** 링크 요청

## 4. 상세 기능 명세

| 기능명 | 기능 설명 | 우선순위 |
| :--- | :--- | :---: |
| 소셜 로그인 연동 | Kakao, Naver, Google 등 주요 소셜 계정 연동 가입 및 로그인 | High |
| 이메일 회원가입 | 이메일 중복 체크, 비밀번호 복잡도 검증 및 계정 생성 | High |
| JWT 기반 인증 | 세션 기반이 아닌 토큰(JWT) 방식의 인증으로 모바일 확장성 확보 | High |
| 아이디/비밀번호 찾기 | 이메일 인증을 통한 임시 비밀번호 발급 또는 재설정 기능 | Medium |
| 자동 로그인 및 보안 | Refresh Token을 활용한 자동 로그인 및 로그아웃 기능 | Medium |

## 5. 비기능적 요구사항
- **보안**: 비밀번호는 반드시 단방향 해시(BCrypt 등)로 암호화하여 저장
- **규제 준수**: 개인정보 보호법에 따른 수집 항목 고지 및 동의 절차 포함
- **성능**: 응답 시간 1초 이내의 빠른 인증 처리 속도 유지
- **안정성**: 동시 접속자가 많을 경우에도 인증 서버의 가용성 확보
