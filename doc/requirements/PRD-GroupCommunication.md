# [PRD] 그룹 소통 및 멤버 태깅

## 1. 개요 및 목적
여행 그룹 내 멤버들 간의 원활한 소통을 위해 실시간 메시징(또는 댓글) 환경에서 특정 인물을 지칭하고 알림을 보낼 수 있는 기능을 제공합니다.

## 2. 타겟 사용자 프로파일
- 그룹 여행 중 특정 멤버에게 빠르게 의사를 전달해야 하는 사용자
- 여러 명의 대화 중 자신을 부르는 메시지를 놓치고 싶지 않은 사용자

## 3. 핵심 사용자 여정 (User Flow)
1. **그룹 대화방** 또는 게시물 **댓글창** 진입
2. 입력창에 **'@'** 입력 시 그룹 멤버 리스트 자동 팝업
3. 원하는 **멤버 선택** 후 메시지 작성
4. 메시지 전송 시 태그된 멤버에게 **실시간 푸시 알림** 발송
5. 태그된 멤버는 알림을 클릭하여 **해당 메시지 위치로 이동**

## 4. 상세 기능 명세

| 기능명 | 기능 설명 | 우선순위 |
| :--- | :--- | :---: |
| 멤버 자동완성 | '@' 기반의 그룹 멤버 검색 및 선택 UI 제공 | High |
| 멘션 알림 | 태깅된 멤버에게 즉각적인 앱 내/푸시 알림 전송 | High |
| 메시지 공유 | 특정 게시물이나 경로 링크를 그룹 메시지로 전송 | Medium |
| 실시간 채팅 | 그룹 멤버들만 참여하는 폐쇄형 대화 공간 | Medium |

## 5. 비기능적 요구사항
- **실시간성**: 멘션 발생 시 알림 지연 시간 2초 이내 유지
- **UI/UX**: 많은 인원 태깅 시 가독성을 해치지 않는 멘션 스타일링(색상 강조 등) 적용
- **권한**: 그룹 멤버가 아닌 사용자는 태깅 대상에서 제외 처리
