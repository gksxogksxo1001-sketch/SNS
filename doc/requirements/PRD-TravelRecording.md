# [PRD] 여행 기록 및 소셜 인터랙션

## 1. 개요 및 목적
사용자가 여행 중의 순간을 쉽고 빠르게 기록하고, 태그(#, @)를 통해 위치 정보를 연결하며, 다른 사용자들과 좋아요 및 댓글로 소통할 수 있는 기능입니다.

## 2. 타겟 사용자 프로파일
- 여행지에서 실시간으로 기록을 남기고 싶은 2030 여행자
- 복잡한 위치 설정보다 익숙한 SNS 방식(태그)의 입력 환경을 선호하는 사용자

## 3. 핵심 사용자 여정 (User Flow)
1. **게시물 작성 버튼** 클릭
2. **사진/텍스트** 입력
3. 본문에 **#강남역** 또는 **@에펠탑**과 같이 장소 태그 입력
4. 태그된 위치 정보가 게시물 하단/지도에 **자동 표시**됨을 확인
5. 게시물 **업로드**
6. 다른 사용자의 게시물에 **좋아요/댓글** 남기기

## 4. 상세 기능 명세

| 기능명 | 기능 설명 | 우선순위 |
| :--- | :--- | :---: |
| 퀵 포스팅 | 사진 및 텍스트 중심의 빠른 게시물 작성 인터페이스 | High |
| 비용 입력 필드 | 비행기, 이동 수단, 숙박비 등을 개별적으로 입력할 수 있는 필드 | High |
| 태그형 위치 인식 | #, @를 사용한 텍스트 기반 장소 검색 및 자동 지도 맵핑 | High |
| 공개 범위 설정 | 전체 공개, 친구 공개, 그룹 공개 등 게시물 노출 권한 관리 | Medium |
| 인터랙션 | 게시물 단위 좋아요 및 댓글 작성/수정/삭제 | Medium |
| 외부 공유 | 게시물 URL 복사 및 타 SNS 공유 기능 | Low |

## 5. 비기능적 요구사항
- **성능**: 태그 입력 시 실시간으로 위치 후보군을 보여주는 자동완성 추천 성능 최적화
- **정확도**: 텍스트 태그가 실제 지도 데이터와 매칭되는 정확도 확보
- **UX**: SNS 특유의 반응형 디자인 적용으로 모바일에서의 게시물 작성 편의성 극대화
