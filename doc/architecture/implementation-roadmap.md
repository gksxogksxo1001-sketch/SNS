# 구현 로드맵 및 모듈화 전략 (Implementation Roadmap)

## 1. 개요
본 문서는 100명 이내의 소규모 사용자를 대상으로 하는 SNS 서비스의 단계별 구현 계획을 정의합니다. 초기 MVP(Minimum Viable Product)는 서비스의 핵심 가치인 '계정', '게시물', '그룹' 기능에 집중합니다.

## 2. 개발 단계별 마일스톤

### [Step 1] MVP 핵심 기반 구축 (우선순위: High)
*목표: 사용자가 가입하고, 사진을 올리고, 그룹을 만들 수 있는 환경 조성*
1. **인증 및 프로필**: Firebase Auth 연동 및 기본 프로필 설정 화면
2. **소셜 피드**: 게시물 작성(Firebase Storage 저장) 및 리스트 조회 기능
3. **기본 그룹 관리**: 그룹 생성 및 멤버 초대 (NoSQL 문서 기반)

### [Step 2] 소셜 인터랙션 및 고도화 (우선순위: Medium)
*목표: 사용자 간의 소통 강화*
1. **인터랙션**: 좋아요 및 댓글 기능 구현
2. **인원 태깅**: 그룹 메시지/댓글 내 @닉네임 멘션 및 알림(FCM) 발송
3. **공유 기능**: 게시물 외부 링크 공유

### [Step 3] 여행 특화 기능 확장 (Phase 2)
*목표: 여행 데이터 시각화 및 비용 관리*
1. **지도 연동**: 게시물 태그 기반의 이동 경로 시각화
2. **비용 정산**: 지출 내역 합산 및 더치페이 자동 계산기
3. **경로 효율 분석**: 장소 간 거리 정보를 활용한 최적화 가이드

## 3. 모듈화 및 인터페이스 전략
- **Firebase SDK Service**: DB 연동(Firestore), 인증(Auth), 파일(Storage) 로직을 프론트엔드 컴포넌트와 분리하여 공통 서비스 모듈로 관리
- **Component-Driven Develop**: 회원가입, 포스 트 카드, 그룹 멤버 리스트 등 UI 요소를 모듈화하여 재사용성 확보
- **Serverless API Layer**: 복잡한 정산 계산이나 보안이 필요한 로직은 Next.js API Routes로 분산하여 처리

## 4. 규모 및 비용 고려 사항
- **사용자 수 (100명 이내)**: Firebase Spark Plan의 무료 티어 내에서 충분히 수용 가능하며, DB 읽기/쓰기 횟수 최적화를 통해 무료 운영 지속 가능
- **성능**: 100명 규모에서는 별도의 로드 밸런서나 복잡한 캐싱 없이도 충분한 응답 속도 확보 가능
