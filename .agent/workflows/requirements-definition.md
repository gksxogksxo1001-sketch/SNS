# 요구사항 정의 워킹플로우 (Requirements Definition Workflow)

이 워크플로우는 사용자의 모호한 아이디어를 구체화하여 전문적인 요구사항 명세서(PRD)를 작성하는 과정을 안내합니다.

## 진행 단계 (Step-by-Step Process)

### 1단계: 요구사항 도출 인터뷰
사용자의 아이디어를 구체화하기 위해 심층 인터뷰를 진행합니다.
- **활동**: 한 번에 1~2개의 핵심 질문을 던지며 사용자의 의도를 파악합니다.
- **주요 질문 내용**:
    - 타겟 사용자층
    - 핵심 가치 및 해결하려는 문제
    - 반드시 포함되어야 할 필수 기능
    - 벤치마킹 대상/유사 서비스
- **종료 조건**: 기능 도출을 위한 충분한 정보가 수집되었을 때

### 2단계: 요구사항 리스트 작성 및 고객 리뷰
인터뷰 내용을 바탕으로 기능 단위의 요구사항 리스트를 작성합니다.
- **출력물**: `doc/requirements/requirements-list.md`
- **형식**: 서술어를 사용하여 한 문장에 하나의 기능만 설명
- **확인 사항**: 사용자에게 리스트를 보여주고 추가/수정/삭제 피드백을 받아 최종 컨펌 진행

### 3단계: 최종 요구사항 명세서(PRD) 작성
승인된 요구사항 리스트를 바탕으로 상세 명세서를 작성합니다.
- **사전 작업**: 프로젝트 개요(Project Overview) 문서 작성 및 공유
- **출력물**: `doc/requirements/PRD-[기능명].md` (기능 단위로 분할 작성)
- **명세서 포함 항목**:
    1. 프로젝트 개요 및 목적
    2. 타겟 사용자 프로파일
    3. 핵심 사용자 여정 (User Flow)
    4. 상세 기능 명세 (우선순위 포함)
    5. 비기능적 요구사항 (성능, 보안 등)

## 출력 파일 경로 규칙
- 요구사항 리스트: `doc/requirements/requirements-list.md`
- 요구사항 명세서: `doc/requirements/PRD-[기능명].md`
