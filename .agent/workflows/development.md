# 개발 및 구현 워크플로우 (Development & Implementation Workflow)

이 워크플로우는 설계된 내용을 바탕으로 실제 기능을 단계적으로 구현하는 과정을 안내합니다. 안정적인 기반을 먼저 구축한 뒤 기능을 확장하는 방식을 따릅니다.

## 단계별 구현 원칙

1. **기능 단위 구현**: 각 마일스톤은 독립적으로 동작 가능한 기능 단위로 쪼개어 배포 가능한 상태를 유지합니다.
2. **반복적 확장**: 핵심 기능을 먼저 구현하고, 사용자 피드백이나 필요에 따라 점진적으로 고도화합니다.

---

## [Phase 1] 핵심 인증 시스템 구축 (Authentication)
사용자가 서비스에 안전하게 진입할 수 있는 최소한의 장치를 마련합니다.

1. **Firebase Auth 설정**: 프로젝트 내 Firebase SDK 초기화 및 제공자(Email, Google, Kakao, Naver) 활성화.
2. **회원가입 로직**: 이메일, 비밀번호와 더불어 **휴대폰 본인 확인 절차**를 포함한 가입 기능 구현.
3. **로그인/로그아웃**: 세션 관리 및 인증 상태에 따른 페이지 접근 제어(Middleware) 및 자동 로그인 구현.

---

## [Phase 2] 데이터 CRUD 엔진 구현 (Data Management)
서비스의 핵심인 여행 기록 데이터를 처리하는 비즈니스 로직을 구축합니다.

1. **Create (생성)**: 게시물 작성, 이미지 업로드(Storage), **Geopoint 기반 장소 저장** 및 태그(#) 처리 로직.
2. **Read (조회)**: 메인 피드 및 개인 프로필의 여행 아카이브(지도/그리드) 리스트 및 상세 조회.
3. **Update (수정)**: 게시물 내용 수정, **지출 항목(Expenses)** 정보 및 정산 상태 업데이트.
4. **Delete (삭제)**: 게시물 및 관련 하위 데이터(댓글, 이미지, 개별 지출 내역)의 안전한 삭제.

---

## [Phase 3] UI 레이아웃 및 디자인 시스템 적용 (UI/UX)
기능이 동작하는 로직 위에 설계된 디자인 시스템을 입히고 사용자 인터페이스를 정밀하게 구성합니다.

1. **Global Layout**: Header, Navigation 등 공통 레이아웃 구성.
2. **Component Styling**: `design-system.md` 가이드에 따른 색상, 폰트, 버튼 스타일 적용 (Tailwind CSS 활용).
3. **Responsive Design**: 모바일 우선(Mobile-First) 반응형 레이아웃 최적화.

---

## [Phase 4] 모듈 단위 기능 확장 (Modular Expansion)
핵심 기반이 완성된 후, 추가 기능을 하나씩 독립된 모듈로 개발하여 붙여나갑니다.

1. **그룹 및 커뮤니티**: 그룹 생성/참여, 멤버 초대, 실시간 메시지 및 댓글 내 **인원 태깅(@닉네임)** 기능.
2. **지도 기반 시각화**: Mapbox/Google Maps API를 이용한 **경로 시각화(Polyline)** 및 스와이프 카드 연동.
3. **정산 및 효율 분석**: 
    - **정산 대시보드**: 그룹별 총 지출 합계 및 나의 분담금 자동 계산기 구현.
    - **경로 효율 가이드**: 장소 간 거리/시간 분석 기반의 최적화 안내 메시지 제공.
4. **알림 시스템**: 좋아요, 댓글, 멘션 및 **정산 요청**에 대한 실시간 푸시 알림(FCM).
